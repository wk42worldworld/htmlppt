# 사람을 위한 튜토리얼

이 튜토리얼은 코드를 쓰지 않고 `.ppt.html` 프레젠테이션을 만들고 싶은 사용자를 위한 안내입니다. 완성된 파일은 발표, 공유, 재편집이 가능합니다.

## 목차

1. PPT.html 작업 흐름 이해
2. 편집기 열기
3. 템플릿에서 시작
4. 문서 정보와 테마 수정
5. 슬라이드 내용 수정
6. 슬라이드 순서 관리
7. 이미지, 데이터, 차트, 표, 코드 추가
8. AI JSON으로 초안 만들기
9. 검사 보고서로 문제 수정
10. 발표하기
11. 저장, 공유, 다시 편집
12. 문제 해결

## 1. PPT.html 작업 흐름 이해

PPT.html Studio는 프레젠테이션을 세 층으로 나눕니다.

- 내용: 제목, 부제목, 본문, 목록, 이미지, 지표, 차트, 표, 코드.
- 레이아웃: 표지, 비교, 카드, 데이터, 차트, 표 같은 안정적인 템플릿.
- 재생: 구조화된 내용을 브라우저에서 재생 가능한 HTML 슬라이드로 렌더링.

HTML을 직접 쓸 필요가 없습니다. 기존 슬라이드 도구처럼 수많은 텍스트 박스를 손으로 움직일 필요도 없습니다. 내용을 편집하고, 레이아웃을 선택하고, 검사한 뒤 하나의 `.ppt.html` 파일로 저장합니다.

## 2. 편집기 열기

두 가지 방식으로 사용할 수 있습니다.

데스크톱 앱:

1. GitHub Releases에서 운영체제에 맞는 패키지를 다운로드합니다.
2. PPT.html Studio를 실행합니다.
3. 열기, 저장, 다른 이름으로 저장을 지원하므로 실제 편집에는 데스크톱 앱을 권장합니다.

브라우저 개발 버전:

```bash
npm install
npm run serve
```

다음을 엽니다.

```text
http://localhost:5173
```

브라우저 버전은 미리보기와 개발 확인에 유용합니다. 저장 시 새 파일을 다운로드합니다.

## 3. 템플릿에서 시작

상단의 `模板` 을 클릭하고 템플릿을 선택합니다.

- AI 감독 카메라: 제품 demo 예시.
- 제품 피치: 제품 소개, 발표, 피치덱용.
- 강의 자료: 수업과 교육 자료용.
- 프로젝트 보고: 상태, 마일스톤, 위험, 의사결정용.

템플릿을 선택하면 왼쪽에는 슬라이드 목록, 가운데에는 현재 슬라이드, 오른쪽에는 편집 가능한 필드가 표시됩니다.

처음 사용할 때는 `프로젝트 보고`가 좋습니다. 표지, 데이터, 표, 비교, 마무리 슬라이드를 한 번에 확인할 수 있습니다.

## 4. 문서 정보와 테마 수정

오른쪽 패널의 `文稿` 영역에서 전체 설정을 수정합니다.

- 이름: 저장할 때 기본 파일명으로 사용됩니다.
- 테마: 전체 문서의 시각 스타일을 정합니다.

사용 가능한 테마:

- `Paper`: 밝고 깔끔함. 강의와 문서형 보고에 적합합니다.
- `Launch`: 어둡고 대비가 강함. 발표와 demo에 적합합니다.
- `Studio`: 현대적인 도구 느낌. 제품과 디자인 제안에 적합합니다.
- `Boardroom`: 공식적이고 절제된 느낌. 비즈니스 보고에 적합합니다.

테마를 바꾸면 가운데 캔버스가 즉시 업데이트됩니다.

## 5. 슬라이드 내용 수정

왼쪽에서 슬라이드를 클릭합니다. 오른쪽 `页面` 영역에 현재 슬라이드의 필드가 표시됩니다.

- 레이아웃: 슬라이드 배치를 결정합니다.
- 키커: `Problem`, `Plan`, `Demo` 같은 작은 라벨.
- 제목: 슬라이드에서 가장 중요한 한 문장.
- 부제목: 보조 설명.
- 본문: 긴 설명.
- 발표 메모: 발표자가 보는 메모.

레이아웃에 따라 표시되는 필드가 달라집니다.

- `对比`: 왼쪽과 오른쪽 제목 및 본문.
- `三卡片`: 카드 내용.
- `数据`: 지표.
- `图表`: 차트 종류, 라벨, 시리즈, 단위.
- `表格`: 열과 행.
- `代码`: 코드 입력란.

편집 팁:

- 한 슬라이드에는 하나의 핵심만 담습니다.
- 제목은 짧게 씁니다.
- 목록은 3개에서 6개로 유지합니다.
- 데이터 슬라이드의 지표는 1개에서 3개로 유지합니다.
- 차트 슬라이드는 라벨 2개에서 8개를 권장합니다.
- 내용이 많으면 여러 슬라이드로 나눕니다.

캔버스에서 직접 편집:

- 캔버스의 제목, 부제목, 본문, 목록 항목, 카드, 지표, 표 셀, 차트 범례, 코드 블록을 더블클릭하면 그 자리에서 텍스트를 수정할 수 있습니다.
- 한 줄 내용은 `Enter` 로 저장합니다. 여러 줄 내용은 줄바꿈을 사용하고 `Ctrl+Enter` 또는 `Cmd+Enter` 로 저장합니다.
- `Esc` 를 누르면 현재 캔버스 편집을 취소합니다.
- 편집 가능한 텍스트나 콘텐츠 블록을 클릭하면 선택 프레임이 표시됩니다.
- 선택한 요소를 드래그하면 가볍게 위치를 조정할 수 있습니다. 선택 프레임의 모서리나 변 핸들을 드래그하면 너비와 높이를 조정할 수 있습니다.
- 요소를 선택한 상태에서 방향키를 누르면 1픽셀씩 미세 조정됩니다. `Shift` 를 누른 채 방향키를 누르면 10픽셀씩 이동합니다.
- 선택 프레임의 `重置` 를 클릭하거나 `Delete` / `Backspace` 를 누르면 해당 요소의 저장된 위치와 크기를 지울 수 있습니다.
- 위치와 크기는 현재 슬라이드의 `canvas` 데이터에 저장되며, 내보낸 `.ppt.html` 파일에서도 유지됩니다.
- 잘못 드래그했을 때는 `撤销` 로 되돌릴 수 있습니다.
- 드래그는 작은 보정에 사용하는 것을 권장합니다. 큰 레이아웃 변경은 레이아웃을 바꾸거나 슬라이드를 나누는 편이 AI와 사람이 계속 편집하기 좋습니다.

## 6. 슬라이드 순서 관리

왼쪽 슬라이드 목록에서 다음을 할 수 있습니다.

- `+`: 현재 슬라이드 뒤에 새 슬라이드 추가.
- `复制`: 현재 슬라이드 복제.
- `上移` / `下移`: 순서 변경.
- `删除`: 현재 슬라이드 삭제.
- 썸네일 클릭: 슬라이드 전환.
- 썸네일 드래그: 슬라이드 순서 직접 변경.

슬라이드가 1장뿐이면 삭제 버튼은 비활성화됩니다.

## 7. 이미지, 데이터, 차트, 표, 코드 추가

이미지:

1. 레이아웃을 `封面`, `左文右图`, `左图右文`, `全屏图片`, 또는 `背景图文` 으로 바꿉니다.
2. `选择本地图片` 를 클릭합니다.
3. 이미지는 Data URI로 `.ppt.html` 파일 안에 포함됩니다.
4. `适配` 으로 채우기 crop 또는 전체 표시를 선택합니다.
5. 대체 텍스트와 선택적 캡션을 추가할 수 있습니다.

이미지 파일을 가운데 캔버스로 직접 드래그할 수도 있습니다. 현재 슬라이드가 이미지 레이아웃이 아니면 앱이 자동으로 `左文右图` 로 바꾸고 이미지를 현재 `.ppt.html` 파일에 포함합니다.

데이터 슬라이드:

`数据` 레이아웃에서는 한 줄에 하나의 지표를 씁니다.

```text
72% | 진행률 | 핵심 흐름 완료
2 | 주요 위험 | 리소스와 출시 일정
1 | 결정 사항 | 파일럿 확대 여부
```

차트 슬라이드:

레이아웃을 `图表` 로 바꾸고 막대, 선, 도넛 차트를 선택합니다.

라벨은 `|` 로 구분합니다.

```text
Q1 | Q2 | Q3 | Q4
```

시리즈는 한 줄에 하나씩 입력합니다.

```text
매출 | 12 | 20 | 31 | 42
비용 | 8 | 11 | 18 | 24
```

도넛 차트는 첫 번째 시리즈를 조각으로 사용합니다.

표:

열 이름은 `|` 로 구분합니다.

```text
단계 | 시간 | 상태
```

행도 같은 형식입니다.

```text
요구사항 확인 | 1주차 | 완료
프로토타입 | 2주차 | 진행 중
파일럿 | 4주차 | 시작 전
```

코드:

레이아웃을 `代码` 로 바꾸고 코드 필드에 붙여넣습니다.

## 8. AI JSON으로 초안 만들기

상단의 `AI JSON` 을 클릭하고 AI가 생성한 deck JSON을 붙여넣은 뒤 `从文本载入` 을 클릭합니다.

PPT.html Studio는 일반 JSON을 읽을 수 있습니다.

```json
{
  "version": "0.1",
  "title": "내 프레젠테이션",
  "theme": "launch",
  "aspectRatio": "16:9",
  "slides": [
    {
      "id": "slide-1",
      "layout": "hero",
      "title": "첫 슬라이드",
      "subtitle": "이것은 PPT.html 입니다"
    }
  ]
}
```

`json` 코드 블록도 그대로 붙여넣을 수 있습니다.

````markdown
```json
{
  "version": "0.1",
  "title": "내 프레젠테이션",
  "theme": "launch",
  "slides": []
}
```
````

AI에게 줄 수 있는 지시 예시:

```text
PPT.html Studio용 deck JSON을 생성해 주세요.
주제: 프로젝트 진행 보고
분량: 5장
톤: 공식적이고 간결하게
version 0.1을 사용하고 자유 형식 HTML은 출력하지 마세요.
```

## 9. 검사 보고서로 문제 수정

AI JSON을 가져온 뒤 상단의 `检查` 를 클릭합니다.

검사 보고서는 다음을 알려줍니다.

- 공유 가능한지 여부.
- 오류, 경고, 팁의 개수.
- 수정이 필요한 필드.
- AI가 어떻게 수정해야 하는지.

`ERROR` 가 있으면:

1. `复制报告` 를 클릭합니다.
2. 보고서를 AI에게 붙여넣습니다.
3. “보고서에 따라 수정하고 전체 deck JSON을 반환해 주세요”라고 요청합니다.
4. 수정된 JSON을 `AI JSON` 패널에서 다시 가져옵니다.

`WARNING` 또는 `TIP` 만 있으면 보통 재생은 가능하지만 개선하는 것이 좋습니다.

## 10. 발표하기

상단의 `演示` 를 클릭합니다. 데스크톱 앱은 전체 화면 발표로 들어가고, 브라우저 버전은 브라우저가 허용할 때 전체 화면을 요청합니다. 브라우저가 차단하면 플레이어 안의 `全屏` 을 클릭합니다.

발표 중에는:

- `F5` 로 첫 슬라이드부터 시작합니다.
- `Shift+F5` 로 현재 슬라이드부터 시작합니다.
- `上一页` / `下一页` 버튼을 클릭합니다.
- 화살표 키, `PageUp`, `PageDown` 으로 이동합니다.
- Space, `Enter`, `N` 으로 다음 슬라이드로 이동합니다.
- `P` 또는 `Backspace` 로 이전 슬라이드로 돌아갑니다.
- `Home` / `End` 로 첫 슬라이드 또는 마지막 슬라이드로 이동합니다.
- `退出` 을 클릭하거나 `Esc` 로 편집기로 돌아갑니다.

발표 컨트롤은 마우스가 멈추면 자동으로 숨겨집니다. 마우스를 움직이면 다시 표시됩니다.

공유하기 전에 처음부터 끝까지 한 번 재생해 보는 것이 좋습니다.

## 11. 저장, 공유, 다시 편집

브라우저 버전:

- `保存 / 下载` 을 클릭합니다.
- 브라우저가 `.ppt.html` 파일을 다운로드합니다.

데스크톱 앱:

- `保存`: 현재 파일에 저장.
- `另存为`: 새 위치에 저장.
- 단축키: `Ctrl/Cmd+S`.

공유:

- `.ppt.html` 파일을 다른 사람에게 보냅니다.
- 상대방은 브라우저에서 바로 열어 발표할 수 있습니다.
- PPT.html Studio에서 다시 열어 편집할 수도 있습니다.

## 12. 문제 해결

### 파일을 더블 클릭해서 열 수 있나요?

네. `.ppt.html` 은 HTML 파일이므로 브라우저에서 열 수 있습니다.

### 왜 PPTX를 직접 생성하지 않나요?

PPTX는 내부 구조가 복잡해 AI가 안정적으로 만들기 어렵고 다시 편집하기도 쉽지 않습니다. PPT.html은 구조화된 JSON을 사용하므로 AI와 사람이 함께 유지하기 좋습니다.

### AI 생성 내용을 가져오지 못하면?

다음을 확인하세요.

- JSON이 완전한지.
- `version` 이 `"0.1"` 인지.
- `slides` 가 배열인지.
- 모든 슬라이드에 `layout` 과 `title` 이 있는지.

그다음 `检查` 를 사용하거나 오류 메시지를 AI에게 전달해 수정합니다.

### 이미지가 사라지지 않나요?

`选择本地图片` 로 선택한 이미지는 `.ppt.html` 파일에 포함됩니다. 이미지 URL을 직접 입력한 경우에는 받는 사람이 그 URL에 접근할 수 있어야 합니다.

### 언제 슬라이드를 나누어야 하나요?

제목이 길거나, 목록이 6개를 넘거나, 핵심 지표가 3개를 넘거나, 표가 너무 넓으면 슬라이드를 나누는 것이 좋습니다.

## 완료 전 체크리스트

- 모든 슬라이드에 명확한 제목이 있습니다.
- 슬라이드 순서가 이야기 흐름에 맞습니다.
- 지표와 표에 임시 내용이 남아 있지 않습니다.
- `检查` 를 실행했습니다.
- 전체 발표를 한 번 재생했습니다.
- `.ppt.html` 로 저장했습니다.
