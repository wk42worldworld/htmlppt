# PPT.html Studio 路线图

这份路线图的目标是让 PPT.html Studio 从可发布 MVP 逐步变成可长期使用的 AI 时代演示工具。

## 产品原则

- AI 负责生成结构化内容，不直接写自由 HTML。
- 人类通过可视化编辑器修改内容、顺序、图片和模板。
- `.ppt.html` 保持单文件、可播放、可再次编辑。
- 所有高级能力都优先服务“可逆编辑”：AI 改完，人类还能改；人类改完，AI 还能继续理解。

## 阶段 1：可信工作流

状态：已开始执行。

目标：

- 让用户可以从模板起稿。
- 让桌面端具备打开、保存、另存为。
- 让 AI JSON 导入具备检查和修复闭环。
- 让普通用户可以把本地图片放进 deck。

已完成：

- 模板：AI 导演相机、产品发布、课程课件、项目汇报。
- 文件：Electron 桥接打开、保存、另存为。
- 检查：PPT.html Validation Report，包含 error、warning、tip 和 AI 修复指令。
- 导入：支持直接粘贴带 `json` 围栏的代码块。
- 图片：本地图片可转为 Data URI 写入当前页面。
- 测试：新增 smoke test，覆盖模板、校验报告和 fenced JSON。

验收标准：

- 普通用户 3 分钟内能从模板生成并保存一份 `.ppt.html`。
- AI 生成 JSON 后，用户能通过检查报告让 AI 修复问题。
- 四平台安装包仍能由 GitHub Actions 自动构建。

## 阶段 2：表达能力

目标：

- 补齐常见演示版式。
- 让非设计用户也能做出完整汇报。

候选功能：

- Agenda / 目录页。
- Roadmap / 路线图页。
- Image Grid / 多图展示页。
- Chart / 基础柱状图、折线图、饼图。
- Callout / 标注页。
- Quote + Image / 人物引用页。
- Section progress / 章节进度条。

验收标准：

- 80% 常见商业汇报不需要自由拖拽文本框。
- AI 可以稳定选择 layout 并生成可编辑内容。

## 阶段 3：导出和分发

目标：

- 让 `.ppt.html` 不只是能播放，也能交付给传统场景。

候选功能：

- PDF 导出。
- PNG/JPEG 批量导出。
- 打印优化。
- Release 内提供 checksums。
- macOS / Windows 签名与公证。

验收标准：

- 用户可以把同一份 deck 交付为 HTML、PDF 和图片。
- 安装包减少系统安全提示。

## 阶段 4：AI 深度协作

目标：

- 从“AI 生成初稿”升级为“AI 持续协作编辑”。

候选功能：

- 单页改写提示词。
- 根据检查报告一键生成 AI 修复 prompt。
- Deck diff：显示 AI 修改了哪些页面。
- 结构化备注：为演讲者备注、目标受众、语气、时长建模。
- Prompt packs：产品发布、融资 BP、课程、技术分享、项目汇报。

验收标准：

- AI 能按页修改，不破坏其他页面。
- 用户能理解和接受 AI 修改。

## 阶段 5：工程质量

目标：

- 让项目能稳定增长，而不是越改越脆。

候选功能：

- Playwright 关键路径测试脚本化。
- 视觉回归截图。
- Schema 自动校验。
- Release 前自动构建示例 deck。
- 更多真实示例。

验收标准：

- 每次发布前能自动验证编辑器、播放器、示例、安装包。
- 新增 layout 不会破坏旧 deck。
