# 普通用户教程

这份教程面向不写代码的用户。目标是用 PPT.html Studio 做出一份可以播放、可以分享、可以再次编辑的 `.ppt.html` 演示文稿。

## 1. 理解 PPT.html 工作流

PPT.html Studio 的核心思想是把演示文稿分成三层：

- 内容层：标题、副标题、正文、列表、图片、数据、图表、表格等。
- 版式层：封面、对比、三卡片、数据页、图表页、表格页等固定模板。
- 播放层：软件把结构化内容渲染成可播放的 HTML 幻灯片。

你不需要写 HTML，也不需要像传统 PPT 那样反复拖动文本框。你只需要编辑内容、选择版式、检查结果，然后保存成一个 `.ppt.html` 文件。

## 2. 打开编辑器

有两种使用方式。

桌面版：

1. 在 GitHub Releases 下载适合你系统的安装包。
2. 打开 PPT.html Studio。
3. 推荐桌面版用于正式编辑，因为它支持打开、保存和另存为。

浏览器开发版：

```bash
npm install
npm run serve
```

然后打开：

```text
http://localhost:5173
```

浏览器版适合预览和开发测试。浏览器版保存时会下载一个新文件。

## 3. 从模板创建第一份文稿

点击顶部 `模板`，选择一个起稿模板：

- AI 导演相机：产品 demo 示例，适合理解格式能力。
- 产品发布：适合产品介绍、路演、发布会。
- 课程课件：适合教学内容、培训材料。
- 项目汇报：适合周报、里程碑、风险与决策。

选择模板后，左侧会出现一组页面，中间显示当前页面，右侧显示可编辑字段。

建议第一次使用时先选择 `项目汇报`，因为它包含封面、数据、表格、对比和结束页，能快速看到主要能力。

## 4. 编辑文稿信息和主题

右侧属性面板的 `文稿` 区域用于编辑整份文稿：

- 名称：保存文件时会用于生成默认文件名。
- 主题：控制整份文稿的颜色和视觉风格。

当前支持的主题：

- `Paper`：浅色、干净，适合课程和文档型汇报。
- `Launch`：深色、高对比，适合发布会和演示。
- `Studio`：现代工具感，适合产品和设计方案。
- `Boardroom`：正式、克制，适合项目汇报和商业场景。

修改主题后，中间画布会立即更新。

## 5. 编辑单页内容

左侧点击任意页面，右侧 `页面` 区域会显示当前页字段：

- 版式：决定这一页如何排版。
- 眉标：页面上方的小标签，例如 `Problem`、`Plan`、`Demo`。
- 标题：当前页最重要的一句话。
- 副标题：补充说明。
- 正文：较长解释文字。
- 备注：演讲时给自己看的提示。

不同版式会显示不同字段。比如：

- `对比` 会显示左侧标题、左侧内容、右侧标题、右侧内容。
- `三卡片` 会显示卡片内容。
- `数据` 会显示指标内容。
- `图表` 会显示图表类型、标签、数据系列和单位。
- `表格` 会显示表头和表格行。
- `代码` 会显示代码内容。

编辑建议：

- 每页只讲一个重点。
- 标题尽量短。
- 列表控制在 3 到 6 条。
- 数据页控制在 1 到 3 个指标。
- 图表页建议使用 2 到 8 个标签。
- 如果内容太多，拆成多页。

画布直观编辑：

- 双击画布上的标题、副标题、正文、列表项、卡片、指标、表格单元格、图表图例或代码块，可以直接在原位置修改文字。
- 单行内容按 `Enter` 保存；多行内容可以换行，按 `Ctrl+Enter` 或 `Cmd+Enter` 保存。
- 按 `Esc` 取消本次画布编辑。
- 单击画布上的可编辑文字或内容块，会显示选中框。
- 选中后，右侧 `元素样式` 面板会启用。你可以调整字号、文字颜色、对齐、加粗、斜体、背景色、边框色、边框粗细、圆角和透明度。
- 拖动被选中的元素可以做轻量位置微调；拖动选中框四角或四边手柄可以调整宽度和高度。
- 选中元素后按方向键可以 1 像素微调；按住 `Shift` 再按方向键可以 10 像素移动。
- 点击选中框上的 `重置`，或按 `Delete` / `Backspace`，可以清除当前元素的位置和尺寸调整。
- 这些位置和尺寸会保存到当前页的 `canvas` 数据里，导出的 `.ppt.html` 也会保持效果。
- 元素样式会保存到当前页的 `styles` 数据里。点击 `清除样式` 只会恢复视觉样式，不会删除文字内容或位置调整。
- 如果拖错了，点击 `撤销` 即可恢复。
- 建议只用拖拽做小范围微调。大幅度改版时，优先切换版式或拆成多页，这样更适合 AI 和人类继续编辑。

## 6. 管理页面顺序

左侧页面栏可以完成常见页面管理：

- `+`：在当前页后添加新页面。
- `复制`：复制当前页。
- `上移` / `下移`：调整顺序。
- `删除`：删除当前页。
- 点击缩略图：切换到对应页面。
- 拖动缩略图：直接调整页面顺序。

如果只剩一页，删除按钮会禁用，避免空文稿。

## 7. 添加图片、视频、音频、数据、图表、表格和代码

主界面中间画布左侧有 `插入` 组件栏。可以直接点击组件，也可以把组件拖到画布上：

- `文本`：在当前页新增一个可见文本框；可以双击修改，也可以拖到画布上的指定位置。
- `图片`：会新增图片页并打开本地图片选择；取消后仍保留图片占位页，方便继续填写 URL 或稍后再选文件。
- `视频`：会新增视频页并打开本地视频选择；取消后仍保留视频占位页。
- `音频`：会新增音频页并打开本地音频选择；取消后仍保留音频占位页。
- `前后`、`决策`：会新增对比页，用于展示改版前后或两个方案取舍。
- `柱状`、`趋势`、`占比`：会新增对应图表页，分别适合类别对比、趋势变化和结构比例。
- `项目表`、`对比表`、`检查表`：会新增不同表格页，用于计划、方案比较或上线检查。
- `三要点`、`功能`、`优劣`、`KPI`、`进度`、`路线图`、`流程`、`里程碑`、`引用`、`代码`：会在当前页后新增对应版式页并填入可编辑示例，不会覆盖当前页内容。

插入后，可以在右侧属性面板继续改内容，也可以在画布上双击文字、拖拽组件、缩放组件。

如果选中的是画布插入对象，右侧 `对象` 面板会先显示位置、尺寸和层级。图片、视频、音频、图表和表格对象还会显示类型化编辑区：

- 媒体对象：直接修改媒体 URL / Data URI、说明、替代文字、封面和适配方式。
- 图表对象：直接修改图表类型、标签、数据系列和单位。
- 表格对象：直接修改表头和每行单元格，也可以用加行、删行、加列、删列按钮。

底部仍保留 `对象数据 JSON（高级）`，适合 AI 或高级用户批量修改对象数据。普通用户优先使用上面的类型化字段。

图片：

1. 把版式切换为 `封面`、`左文右图`、`左图右文`、`全屏图片` 或 `背景图文`。
2. 点击 `选择本地图片`。
3. 选择图片后，图片会以 Data URI 形式写入 `.ppt.html` 文件。
4. 用 `适配` 选择填充裁切或完整显示。
5. 可填写替代文字和图片说明。

也可以直接把图片文件拖到中间画布。当前页如果不是图片版式，软件会自动切换为 `左文右图`，并把图片嵌入当前 `.ppt.html`。

视频和音频：

1. 点击左侧 `视频` 或 `音频`，选择本地文件。
2. 软件会把媒体转成 Data URI，保存时写入同一个 `.ppt.html`。
3. 也可以在右侧填写 URL；保存或下载前，软件会尝试把可访问的 URL 打包进单文件。读取失败时会阻止保存并列出具体资源。

数据页：

在 `数据` 版式中，每行一个指标：

```text
72% | 完成度 | 核心流程已打通
2 | 主要风险 | 资源和上线窗口
1 | 待决策 | 是否扩大试点
```

图表页：

把版式切换为 `图表`，然后选择柱状图、折线图或环形图。

标签用 `|` 分隔：

```text
Q1 | Q2 | Q3 | Q4
```

数据系列每行一个：

```text
收入 | 12 | 20 | 31 | 42
成本 | 8 | 11 | 18 | 24
```

环形图会使用第一行数据系列作为分组。

表格页：

表头用 `|` 分隔：

```text
阶段 | 时间 | 状态
```

表格内容每行一组：

```text
需求确认 | 第 1 周 | 完成
原型验证 | 第 2 周 | 进行中
试点发布 | 第 4 周 | 待开始
```

代码页：

把版式切换到 `代码`，把代码粘贴到代码内容字段即可。

## 8. 使用 AI JSON 起稿

点击顶部 `AI JSON`，把 AI 生成的 deck JSON 粘贴进去，然后点击 `从文本载入`。

PPT.html Studio 支持两种粘贴方式：

纯 JSON：

```json
{
  "version": "0.1",
  "title": "我的演示",
  "theme": "launch",
  "aspectRatio": "16:9",
  "slides": [
    {
      "id": "slide-1",
      "layout": "hero",
      "title": "第一页标题",
      "subtitle": "这是一份 PPT.html"
    }
  ]
}
```

带 `json` 围栏的代码块也可以直接粘贴：

````markdown
```json
{
  "version": "0.1",
  "title": "我的演示",
  "theme": "launch",
  "slides": []
}
```
````

建议给 AI 的指令：

```text
请为 PPT.html Studio 生成 deck JSON。
主题：我的项目汇报
页数：5 页
风格：正式、简洁
必须使用 version 0.1，不要输出自由 HTML。
```

## 9. 使用检查报告修复问题

导入 AI JSON 后，点击顶部 `检查`。

检查报告会告诉你：

- 是否可以正常分享。
- 有多少错误、警告和提示。
- 哪个字段需要修改。
- AI 应该如何修复。

如果报告中出现 `ERROR`：

1. 点击 `复制报告`。
2. 把报告粘贴给 AI。
3. 要求 AI “根据报告修复并输出完整 deck JSON”。
4. 回到 `AI JSON` 面板重新载入。

更省事的方式是点击 `复制 AI 修复 Prompt`。它会把当前 deck JSON、检查报告和修复规则一起复制好，直接粘贴给 AI 即可。

如果只有 `WARNING` 或 `TIP`，文件通常仍然可以播放，但建议继续优化。

## 10. 演示播放

点击顶部 `演示` 进入播放模式。桌面端会进入全屏播放；浏览器版会尽量请求全屏，如果浏览器拦截，可以点击播放画面里的 `全屏`。

在右侧 `文稿` 面板可以设置默认切换动效：`无`、`淡入`、`滑入`、`推入`、`缩放`。在 `页面` 面板可以让单页跟随全局，也可以为某一页单独指定切换动效。导出的单文件播放器会保留这些设置。

播放时可以：

- 按 `F5` 从第一页开始播放。
- 按 `Shift+F5` 从当前页开始播放。
- 点击 `上一页` / `下一页`。
- 使用方向键、`PageUp`、`PageDown` 翻页。
- 按空格、`Enter` 或 `N` 进入下一页。
- 按 `P` 或 `Backspace` 返回上一页。
- 按 `Home` / `End` 跳到第一页或最后一页。
- 点击 `退出` 或按 `Esc` 回到编辑器。

播放控制条会在鼠标静止后自动隐藏，移动鼠标会再次显示。

演示模式显示的是最终观众看到的效果，保存前建议至少完整播放一遍。

## 11. 保存、分享和再次编辑

浏览器版：

- 点击 `保存 / 下载`。
- 浏览器会下载一个 `.ppt.html` 文件。
- 保存前会自动打包外部图片、视频、音频、视频封面和播放器依赖；只要保存成功，分享这一个文件即可。

桌面版：

- `保存`：保存到当前文件。
- `另存为`：选择新的保存位置。
- 快捷键：`Ctrl/Cmd+S`。
- 保存前同样会把外部媒体资源打包进 `.ppt.html`。如果某个 URL 或本地路径不可读取，软件会提示修复后再保存。

分享方式：

- 把 `.ppt.html` 文件发给别人。
- 对方可以直接用浏览器打开播放。
- 对方也可以用 PPT.html Studio 再次打开编辑。

## 12. 常见问题

### 文件可以直接双击播放吗？

可以。`.ppt.html` 本质是 HTML 文件，浏览器可以直接打开。

### 为什么不直接生成 PPTX？

PPTX 内部结构复杂，AI 很容易生成不稳定或难以再次编辑的文件。PPT.html 使用结构化 JSON，更适合 AI 和人类共同维护。

### AI 生成的内容导入失败怎么办？

先确认：

- JSON 是否完整。
- `version` 是否是 `"0.1"`。
- `slides` 是否是数组。
- 每页是否有 `layout` 和 `title`。

然后点击 `检查` 或把错误信息交给 AI 修复。

### 图片会不会丢失？

如果使用 `选择本地图片`，图片会写入 `.ppt.html` 文件，不依赖本地路径。如果手动填 URL，则需要保证接收方能访问该 URL。

### 什么情况下应该拆页？

当一页里出现过长标题、超过 6 条列表、超过 3 个核心指标或表格过宽时，建议拆成多页。

## 完成前检查清单

- 每页标题清楚。
- 页面顺序符合讲述逻辑。
- 数据和表格没有明显占位内容。
- 点击过 `检查`。
- 完整播放过一次。
- 已保存为 `.ppt.html` 文件。
