# 普通用户教程

这份教程面向不写代码的用户。目标是用 PPT.html Studio 做出一份可以播放、可以分享、可以再次编辑的 `.ppt.html` 演示文稿。

## 1. 理解 PPT.html 工作流

PPT.html Studio 的核心思想是把演示文稿分成三层：

- 内容层：标题、副标题、正文、列表、图片、数据、表格等。
- 版式层：封面、对比、三卡片、数据页、表格页等固定模板。
- 播放层：软件把结构化内容渲染成可播放的 HTML 幻灯片。

你不需要写 HTML，也不需要像传统 PPT 那样反复拖动文本框。你只需要编辑内容、选择版式、检查结果，然后保存成一个 `.ppt.html` 文件。

## 2. 打开编辑器

有两种使用方式。

桌面版：

1. 在 GitHub Releases 下载适合你系统的安装包。
2. 打开 PPT.html Studio。
3. 推荐桌面版用于正式编辑，因为它支持打开、保存和另存为。

浏览器开发版：

```bash
npm install
npm run serve
```

然后打开：

```text
http://localhost:5173
```

浏览器版适合预览和开发测试。浏览器版保存时会下载一个新文件。

## 3. 从模板创建第一份文稿

点击顶部 `模板`，选择一个起稿模板：

- AI 导演相机：产品 demo 示例，适合理解格式能力。
- 产品发布：适合产品介绍、路演、发布会。
- 课程课件：适合教学内容、培训材料。
- 项目汇报：适合周报、里程碑、风险与决策。

选择模板后，左侧会出现一组页面，中间显示当前页面，右侧显示可编辑字段。

建议第一次使用时先选择 `项目汇报`，因为它包含封面、数据、表格、对比和结束页，能快速看到主要能力。

## 4. 编辑文稿信息和主题

右侧属性面板的 `文稿` 区域用于编辑整份文稿：

- 名称：保存文件时会用于生成默认文件名。
- 主题：控制整份文稿的颜色和视觉风格。

当前支持的主题：

- `Paper`：浅色、干净，适合课程和文档型汇报。
- `Launch`：深色、高对比，适合发布会和演示。
- `Studio`：现代工具感，适合产品和设计方案。
- `Boardroom`：正式、克制，适合项目汇报和商业场景。

修改主题后，中间画布会立即更新。

## 5. 编辑单页内容

左侧点击任意页面，右侧 `页面` 区域会显示当前页字段：

- 版式：决定这一页如何排版。
- 眉标：页面上方的小标签，例如 `Problem`、`Plan`、`Demo`。
- 标题：当前页最重要的一句话。
- 副标题：补充说明。
- 正文：较长解释文字。
- 备注：演讲时给自己看的提示。

不同版式会显示不同字段。比如：

- `对比` 会显示左侧标题、左侧内容、右侧标题、右侧内容。
- `三卡片` 会显示卡片内容。
- `数据` 会显示指标内容。
- `表格` 会显示表头和表格行。
- `代码` 会显示代码内容。

编辑建议：

- 每页只讲一个重点。
- 标题尽量短。
- 列表控制在 3 到 6 条。
- 数据页控制在 1 到 3 个指标。
- 如果内容太多，拆成多页。

## 6. 管理页面顺序

左侧页面栏可以完成常见页面管理：

- `+`：在当前页后添加新页面。
- `复制`：复制当前页。
- `上移` / `下移`：调整顺序。
- `删除`：删除当前页。
- 点击缩略图：切换到对应页面。

如果只剩一页，删除按钮会禁用，避免空文稿。

## 7. 添加图片、数据、表格和代码

图片：

1. 把版式切换为 `封面`、`左文右图` 或 `左图右文`。
2. 点击 `选择本地图片`。
3. 选择图片后，图片会以 Data URI 形式写入 `.ppt.html` 文件。

数据页：

在 `数据` 版式中，每行一个指标：

```text
72% | 完成度 | 核心流程已打通
2 | 主要风险 | 资源和上线窗口
1 | 待决策 | 是否扩大试点
```

表格页：

表头用 `|` 分隔：

```text
阶段 | 时间 | 状态
```

表格内容每行一组：

```text
需求确认 | 第 1 周 | 完成
原型验证 | 第 2 周 | 进行中
试点发布 | 第 4 周 | 待开始
```

代码页：

把版式切换到 `代码`，把代码粘贴到代码内容字段即可。

## 8. 使用 AI JSON 起稿

点击顶部 `AI JSON`，把 AI 生成的 deck JSON 粘贴进去，然后点击 `从文本载入`。

PPT.html Studio 支持两种粘贴方式：

纯 JSON：

```json
{
  "version": "0.1",
  "title": "我的演示",
  "theme": "launch",
  "aspectRatio": "16:9",
  "slides": [
    {
      "id": "slide-1",
      "layout": "hero",
      "title": "第一页标题",
      "subtitle": "这是一份 PPT.html"
    }
  ]
}
```

带 `json` 围栏的代码块也可以直接粘贴：

````markdown
```json
{
  "version": "0.1",
  "title": "我的演示",
  "theme": "launch",
  "slides": []
}
```
````

建议给 AI 的指令：

```text
请为 PPT.html Studio 生成 deck JSON。
主题：我的项目汇报
页数：5 页
风格：正式、简洁
必须使用 version 0.1，不要输出自由 HTML。
```

## 9. 使用检查报告修复问题

导入 AI JSON 后，点击顶部 `检查`。

检查报告会告诉你：

- 是否可以正常分享。
- 有多少错误、警告和提示。
- 哪个字段需要修改。
- AI 应该如何修复。

如果报告中出现 `ERROR`：

1. 点击 `复制报告`。
2. 把报告粘贴给 AI。
3. 要求 AI “根据报告修复并输出完整 deck JSON”。
4. 回到 `AI JSON` 面板重新载入。

如果只有 `WARNING` 或 `TIP`，文件通常仍然可以播放，但建议继续优化。

## 10. 演示播放

点击顶部 `演示` 进入播放模式。

播放时可以：

- 点击 `上一页` / `下一页`。
- 使用键盘左右方向键。
- 按空格进入下一页。
- 点击 `退出` 回到编辑器。

演示模式显示的是最终观众看到的效果，保存前建议至少完整播放一遍。

## 11. 保存、分享和再次编辑

浏览器版：

- 点击 `保存 / 下载`。
- 浏览器会下载一个 `.ppt.html` 文件。

桌面版：

- `保存`：保存到当前文件。
- `另存为`：选择新的保存位置。
- 快捷键：`Ctrl/Cmd+S`。

分享方式：

- 把 `.ppt.html` 文件发给别人。
- 对方可以直接用浏览器打开播放。
- 对方也可以用 PPT.html Studio 再次打开编辑。

## 12. 常见问题

### 文件可以直接双击播放吗？

可以。`.ppt.html` 本质是 HTML 文件，浏览器可以直接打开。

### 为什么不直接生成 PPTX？

PPTX 内部结构复杂，AI 很容易生成不稳定或难以再次编辑的文件。PPT.html 使用结构化 JSON，更适合 AI 和人类共同维护。

### AI 生成的内容导入失败怎么办？

先确认：

- JSON 是否完整。
- `version` 是否是 `"0.1"`。
- `slides` 是否是数组。
- 每页是否有 `layout` 和 `title`。

然后点击 `检查` 或把错误信息交给 AI 修复。

### 图片会不会丢失？

如果使用 `选择本地图片`，图片会写入 `.ppt.html` 文件，不依赖本地路径。如果手动填 URL，则需要保证接收方能访问该 URL。

### 什么情况下应该拆页？

当一页里出现过长标题、超过 6 条列表、超过 3 个核心指标或表格过宽时，建议拆成多页。

## 完成前检查清单

- 每页标题清楚。
- 页面顺序符合讲述逻辑。
- 数据和表格没有明显占位内容。
- 点击过 `检查`。
- 完整播放过一次。
- 已保存为 `.ppt.html` 文件。
