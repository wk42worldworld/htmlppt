# 人間向けチュートリアル

このチュートリアルは、コードを書かずに `.ppt.html` プレゼンテーションを作りたいユーザー向けです。完成したファイルは再生、共有、再編集できます。

## 1. PPT.html の作成フローを理解する

PPT.html Studio はプレゼンテーションを 3 つの層に分けます。

- 内容：タイトル、サブタイトル、本文、リスト、画像、指標、グラフ、表、コード。
- レイアウト：表紙、比較、カード、データ、グラフ、表などの安定したテンプレート。
- 再生：構造化された内容をブラウザで再生できる HTML スライドに変換。

HTML を書く必要はありません。従来のスライドツールのように、多数のテキストボックスを手作業で動かす必要もありません。内容を編集し、レイアウトを選び、検証し、1 つの `.ppt.html` ファイルとして保存します。

## 2. エディタを開く

使い方は 2 種類あります。

デスクトップアプリ：

1. GitHub Releases から自分の環境に合うパッケージをダウンロードします。
2. PPT.html Studio を起動します。
3. 開く、保存、別名保存を使えるため、正式な編集にはデスクトップアプリを推奨します。

ブラウザ開発版：

```bash
npm install
npm run serve
```

次を開きます。

```text
http://localhost:5173
```

ブラウザ版はプレビューや開発確認に便利です。保存時は新しいファイルをダウンロードします。

## 3. テンプレートから始める

上部の `模板` をクリックし、テンプレートを選びます。

- AI 導演相機：製品 demo の例。
- 製品ピッチ：製品紹介、発表、ピッチ向け。
- 授業資料：授業や研修資料向け。
- プロジェクト報告：進捗、マイルストーン、リスク、意思決定向け。

テンプレートを選ぶと、左にスライド一覧、中央に現在のスライド、右に編集フィールドが表示されます。

初めて使う場合は `プロジェクト報告` がおすすめです。表紙、データ、表、比較、終了スライドをまとめて確認できます。

## 4. デッキ情報とテーマを編集する

右側パネルの `文稿` セクションで全体設定を編集します。

- 名称：保存時の既定ファイル名に使われます。
- テーマ：デッキ全体の見た目を決めます。

利用できるテーマ：

- `Paper`：明るく整理された見た目。授業や文書型レポートに向いています。
- `Launch`：暗色でコントラストが強い見た目。発表や demo に向いています。
- `Studio`：現代的なツール感。製品やデザイン案に向いています。
- `Boardroom`：正式で落ち着いた見た目。ビジネス報告に向いています。

テーマを変更すると、中央のキャンバスがすぐに更新されます。

## 5. スライド内容を編集する

左側で任意のスライドをクリックします。右側の `页面` セクションに現在のスライドの項目が表示されます。

- 版式：スライドの配置を決めます。
- 眉标：`Problem`、`Plan`、`Demo` のような小さなラベル。
- 标题：スライドで最も重要な一文。
- 副标题：補足説明。
- 正文：長めの説明。
- 备注：発表者用メモ。

レイアウトによって表示される項目は変わります。

- `对比`：左右のタイトルと本文。
- `三卡片`：カード内容。
- `数据`：指標。
- `图表`：グラフ種類、ラベル、系列、単位。
- `表格`：列と行。
- `代码`：コード入力欄。

編集のコツ：

- 1 枚につき主題は 1 つ。
- タイトルは短くする。
- リストは 3 から 6 個にする。
- データスライドの指標は 1 から 3 個にする。
- グラフスライドのラベルは 2 から 8 個を目安にします。
- 情報が多い場合はスライドを分ける。

キャンバス上で直接編集：

- キャンバス上のタイトル、サブタイトル、本文、リスト項目、カード、指標、表のセル、グラフ凡例、コードブロックをダブルクリックすると、その場で文字を編集できます。
- 1 行の内容は `Enter` で保存します。複数行の内容は改行を使い、`Ctrl+Enter` または `Cmd+Enter` で保存します。
- `Esc` で現在のキャンバス編集をキャンセルできます。
- 編集可能な文字や内容ブロックをクリックすると、選択枠が表示されます。
- 選択した要素をドラッグすると軽く位置調整できます。選択枠の角または辺のハンドルをドラッグすると、幅と高さを調整できます。
- 要素を選択した状態で矢印キーを押すと 1 ピクセルずつ微調整できます。`Shift` を押しながら矢印キーを押すと 10 ピクセルずつ移動します。
- 選択枠の `重置` をクリックするか、`Delete` / `Backspace` を押すと、その要素の保存済み位置とサイズを消去できます。
- 位置とサイズは現在のスライドの `canvas` データに保存され、書き出した `.ppt.html` でも維持されます。
- 位置を間違えた場合は `撤销` で戻せます。
- ドラッグは小さな調整に使うのがおすすめです。大きく構成を変える場合は、レイアウト変更やスライド分割を優先すると、AI と人間の両方が編集しやすい状態を保てます。

## 6. スライド順を管理する

左側のスライド一覧で操作できます。

- `+`：現在のスライドの後に追加。
- `复制`：現在のスライドを複製。
- `上移` / `下移`：順序を変更。
- `删除`：現在のスライドを削除。
- サムネイルをクリック：スライドを切り替え。
- サムネイルをドラッグ：スライド順を直接変更。

スライドが 1 枚だけの場合、削除は無効になります。

## 7. 画像、動画、音声、データ、グラフ、表、コードを追加する

中央キャンバスの横に `挿入` コンポーネントパレットがあります。コンポーネントはクリックしても、キャンバスへドラッグしても挿入できます。

- `テキスト`：通常のテキストスライドを作成します。
- `画像`：クリックするとローカル画像を選択します。ドラッグすると画像スライドのプレースホルダーを作成します。
- `動画`：クリックするとローカル動画を選択します。ドラッグすると動画スライドのプレースホルダーを作成します。
- `音声`：クリックするとローカル音声を選択します。ドラッグすると音声スライドのプレースホルダーを作成します。
- `グラフ`、`表`、`カード`、`データ`、`タイムライン`、`引用`、`コード`：現在のスライドを対応するレイアウトに切り替え、編集可能な初期内容を入れます。

挿入後は右側パネルで内容を編集できます。キャンバス上で文字をダブルクリックしたり、コンポーネントをドラッグ、リサイズしたりすることもできます。

画像：

1. レイアウトを `封面`、`左文右图`、`左图右文`、`全屏图片`、または `背景图文` にします。
2. `选择本地图片` をクリックします。
3. 画像は Data URI として `.ppt.html` ファイル内に埋め込まれます。
4. `适配` で塗りつぶし裁切または全体表示を選びます。
5. 代替テキストと任意のキャプションを追加できます。

画像ファイルを中央のキャンバスへ直接ドラッグすることもできます。現在のスライドが画像レイアウトでない場合、アプリは自動で `左文右图` に切り替え、画像を現在の `.ppt.html` に埋め込みます。

動画と音声：

1. 挿入パレットの `動画` または `音声` をクリックし、ローカルファイルを選択します。
2. アプリはメディアを Data URI に変換し、同じ `.ppt.html` に保存します。
3. 右側パネルに URL を貼ることもできます。保存またはダウンロード前に、アプリはアクセス可能な URL を単一ファイルへパッケージします。読み込めない場合は保存を止め、該当アセットを表示します。

データスライド：

`数据` レイアウトでは、1 行に 1 つの指標を書きます。

```text
72% | 進捗 | コアフローは完了
2 | 主なリスク | リソースとリリース時期
1 | 意思決定 | 試験運用を拡大するか
```

グラフスライド：

レイアウトを `图表` に変更し、棒グラフ、折れ線グラフ、ドーナツグラフを選びます。

ラベルは `|` で区切ります。

```text
Q1 | Q2 | Q3 | Q4
```

系列は 1 行に 1 つ書きます。

```text
売上 | 12 | 20 | 31 | 42
コスト | 8 | 11 | 18 | 24
```

ドーナツグラフでは最初の系列が分割として使われます。

表：

列名は `|` で区切ります。

```text
段階 | 時期 | 状態
```

行も同じ形式です。

```text
要件確認 | 第 1 週 | 完了
プロトタイプ | 第 2 週 | 進行中
試験運用 | 第 4 週 | 未開始
```

コード：

レイアウトを `代码` に変更し、コード欄に貼り付けます。

## 8. AI JSON で下書きを作る

上部の `AI JSON` をクリックし、AI が生成した deck JSON を貼り付け、`从文本载入` をクリックします。

PPT.html Studio は通常の JSON を読み込めます。

```json
{
  "version": "0.1",
  "title": "私のデッキ",
  "theme": "launch",
  "aspectRatio": "16:9",
  "slides": [
    {
      "id": "slide-1",
      "layout": "hero",
      "title": "最初のスライド",
      "subtitle": "これは PPT.html です"
    }
  ]
}
```

`json` 囲みコードブロックもそのまま貼り付けられます。

````markdown
```json
{
  "version": "0.1",
  "title": "私のデッキ",
  "theme": "launch",
  "slides": []
}
```
````

AI に渡す指示例：

```text
PPT.html Studio 用の deck JSON を生成してください。
テーマ：プロジェクト進捗報告
枚数：5 枚
トーン：正式で簡潔
version 0.1 を使い、自由な HTML は出力しないでください。
```

## 9. 検証レポートで問題を修正する

AI JSON を読み込んだ後、上部の `检查` をクリックします。

検証レポートには次が表示されます。

- 共有可能かどうか。
- エラー、警告、ヒントの数。
- 修正が必要なフィールド。
- AI がどう修正すべきか。

`ERROR` がある場合：

1. `复制报告` をクリックします。
2. レポートを AI に貼り付けます。
3. 「レポートに従って修正し、完全な deck JSON を返してください」と依頼します。
4. 修正版を `AI JSON` パネルで読み込みます。

`WARNING` や `TIP` だけなら、多くの場合は再生できますが、改善する価値があります。

## 10. プレゼンする

上部の `演示` をクリックします。デスクトップアプリでは全画面プレゼンに入り、ブラウザ版ではブラウザが許可する場合に全画面を要求します。ブロックされた場合は、プレイヤー内の `全屏` をクリックします。

発表中の操作：

- `F5` で最初のスライドから開始。
- `Shift+F5` で現在のスライドから開始。
- `上一页` / `下一页` をクリック。
- 矢印キー、`PageUp`、`PageDown` で移動。
- Space、`Enter`、`N` で次へ進む。
- `P` または `Backspace` で前へ戻る。
- `Home` / `End` で最初または最後のスライドへ移動。
- `退出` をクリック、または `Esc` でエディタに戻る。

プレゼン操作バーはマウスが静止すると自動的に隠れます。マウスを動かすと再表示されます。

共有前に最初から最後まで一度再生することをおすすめします。

## 11. 保存、共有、再編集する

ブラウザ版：

- `保存 / 下载` をクリックします。
- `.ppt.html` ファイルがダウンロードされます。
- 保存前に外部画像、動画、音声、動画ポスター、プレイヤー依存ファイルが HTML に同梱されます。保存に成功したら、その 1 ファイルだけ共有すれば十分です。

デスクトップアプリ：

- `保存`：現在のファイルに保存。
- `另存为`：新しい場所を選んで保存。
- ショートカット：`Ctrl/Cmd+S`。
- デスクトップアプリでも同じパッケージ処理を行います。URL またはローカルパスを読めない場合は、そのアセットを修正して再保存します。

共有：

- `.ppt.html` ファイルを相手に送ります。
- 相手はブラウザで直接再生できます。
- PPT.html Studio で再度開いて編集することもできます。

## 12. よくある問題

### ファイルをダブルクリックして開けますか？

はい。`.ppt.html` は HTML ファイルなので、ブラウザで開けます。

### なぜ PPTX を直接生成しないのですか？

PPTX は内部構造が複雑で、AI が安定して生成しにくく、再編集も難しくなりがちです。PPT.html は構造化 JSON を使うため、AI と人間が共同で維持しやすくなります。

### AI 生成内容を読み込めない場合は？

次を確認します。

- JSON が完全か。
- `version` が `"0.1"` か。
- `slides` が配列か。
- 各スライドに `layout` と `title` があるか。

その後、`检查` を使うか、エラー文を AI に渡して修正します。

### 画像は失われませんか？

`选择本地图片` で選んだ画像は `.ppt.html` に埋め込まれます。手動で URL を入力した場合は、受け取る人がその URL にアクセスできる必要があります。

### いつスライドを分けるべきですか？

タイトルが長い、リストが 6 個を超える、主要指標が 3 個を超える、表が広すぎる場合は分けるのがおすすめです。

## 完成前チェックリスト

- すべてのスライドに明確なタイトルがある。
- スライド順が話の流れに合っている。
- 指標や表に仮の内容が残っていない。
- `检查` を実行した。
- 一度通しで再生した。
- `.ppt.html` として保存した。
