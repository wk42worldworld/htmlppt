# HTML 画布编辑项目调研与实施计划

这份文档记录 v0.2.4 画布编辑升级的产品判断：从成熟的开源 HTML/画布编辑器里吸收交互模型，但不把 PPT.html Studio 变成自由网页编辑器。

## 排序结论

排序标准不是单纯 star 数，而是和 PPT.html 的目标匹配程度：AI 友好、人类可编辑、结构可逆、单文件可导出。

1. GrapesJS

   最完整的开源 Web Builder 思路。值得学习组件树、图层、属性面板、资源管理和命令系统。不适合直接整体嵌入，因为它面向自由网页搭建，会把 PPT.html 的结构化优势稀释掉。

2. Puck

   很适合学习“数据驱动的可视化编辑”。它强调用自己的组件和自己的数据模型，正好对应 PPT.html 的核心：AI 写 JSON，人类改 JSON 的可视化投影。

3. Moveable

   最值得直接借鉴交互细节：选中框、拖拽、缩放、旋转、吸附、组合操作。v0.2.4 已经先实现了轻量版的选中框、拖拽、八向缩放和键盘微调。

4. Craft.js

   适合作为 React 页面编辑器架构参考。它的思想有价值，但当前项目是轻量 vanilla JS，不适合为了它引入完整 React 编辑器框架。

5. VvvebJs

   值得学习它的低依赖、块编辑、属性面板做法。它比 GrapesJS 更轻，但仍偏自由 HTML 页面编辑。

6. Selecto

   专注多选框选。暂不引入，但下一阶段做多选、成组移动、批量对齐时可以参考它的选择模型。

7. tldraw / Excalidraw

   都是优秀画布产品，值得学习无限画布、选择态、快捷键、撤销体验。但它们主要编辑图形对象，不是结构化 HTML 幻灯片，所以只借鉴交互，不照搬数据模型。

## 核心产品原则

- 不能让用户自由编辑任意 HTML/CSS，否则 AI 和人类都很难继续维护。
- 每个可编辑元素必须对应一个结构字段路径，例如 `title`、`cards.0.title`、`metrics.1.value`。
- 画布操作只保存轻量几何数据：`x/y/w/h`。
- 大范围排版仍然通过 `layout`、主题、结构字段、拆页完成。
- 导出的 `.ppt.html` 必须保留编辑结果，并且仍然能被软件重新打开。

## v0.2.4 已实现

- 点击可编辑元素显示选中框。
- 拖动元素保存 `canvas[path].x/y`。
- 拖动八个手柄保存 `canvas[path].w/h`。
- 方向键 1 像素微调，`Shift + 方向键` 10 像素微调。
- `Delete` / `Backspace` 或选中框 `重置` 清除当前元素几何调整。
- standalone `.ppt.html` 播放器支持 `x/y/w/h`。
- schema、格式文档、四国语言首页、四国语言普通用户教程、四国语言 AI 教程同步更新。

## 下一阶段计划

### 阶段 1：多选与对齐

- 框选多个元素。
- 按住 `Shift` 增减选择。
- 批量移动。
- 左对齐、右对齐、顶对齐、底对齐、水平居中、垂直居中。
- 等距分布。
- 复制和粘贴 `canvas` 几何设置。

### 阶段 2：吸附与辅助线

- 元素移动时显示中心线、边缘线、页面安全边距线。
- 接近页面中心、边缘、其他元素边缘时自动吸附。
- 提供临时开关：吸附开启 / 关闭。
- 保存时仍只写 `canvas` 数据，不写自由 CSS。

### 阶段 3：图层与可编辑对象列表

- 右侧增加当前页对象列表。
- 点击对象列表可以选中画布元素。
- 显示字段路径和可读名称。
- 支持锁定元素，避免误拖。
- 支持隐藏辅助层，但不隐藏最终播放内容。

### 阶段 4：媒体与复杂组件画布化

- 图片区域支持拖拽裁切焦点。
- 图表组件支持直接编辑标签和数据点。
- 视频、动效、形状、箭头、标注进入结构化组件系统。
- 所有新增组件都必须有 JSON schema，不允许退回自由 HTML。

### 阶段 5：AI 编辑协作

- 给 AI 一个内部系统提示，说明当前 PPT.html schema、当前页字段路径和用户选中的元素。
- AI 可以按局部 patch 修改 deck JSON。
- AI 修改前后生成可审阅 diff。
- 人类可以接受、撤销或继续局部改写。

## 不做的事

- 不把 GrapesJS 这类完整网页搭建器直接塞进软件。
- 不把每个元素都变成任意 CSS 盒子。
- 不优先做 PPTX 级别的无限自由排版。
- 不让 AI 输出不可逆的 HTML。

这个方向的关键不是复制某个项目，而是把成熟画布编辑器的交互手感，压缩进 PPT.html 的结构化格式里。
